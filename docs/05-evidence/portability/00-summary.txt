Context
-------
Post-CI/CD sprint task. The validated Azure DR platform is complete.
This task packages the infrastructure design as cloud-agnostic without
modifying the current Azure implementation.

Critical constraint: Do NOT break the validated Azure platform.
Approach: Additive structure only — new folders alongside existing envs/.

Phase 1 — Current-State Portability Audit
-------------------------------------------
34 components classified across 4 categories:

Cloud-neutral (no changes needed):
  PostgreSQL 16 (streaming replication)
  Keepalived / VIP (on-prem only, not a cloud concern)
  WireGuard (software VPN — runs on any Linux VM)
  FastAPI app / Docker container
  pgBackRest core (repo backend is a config variable)
  All Ansible roles: postgres, postgres_primary, postgres_standby,
    keepalived, wireguard, pgbackrest, app_deploy
  All DR scripts: dr-preflight, onprem-failover, onprem-fallback,
    fullsite-failover, evidence-export

Azure-specific (replaced by provider equivalents):
  Key Vault → AWS Secrets Manager / GCP Secret Manager
  Managed Identity → EC2 IAM profile / GCP service account
  Log Analytics → CloudWatch / Cloud Logging
  Azure Monitor alerts → CloudWatch alarms / GCP alert policies
  VNet + NSG → VPC + security groups / VPC + firewall rules
  AKS → EKS / GKE (dev only — not in DR path)
  ACR → ECR / Artifact Registry (dev only — not in DR path)
  Azure Arc → no equivalent (Azure-specific feature)

Tightly coupled / unsafe to change before presentation:
  envs/dr-fce/ — validated DR environment
  envs/dev/, envs/swe-aks/ — live
  modules/network/, modules/keyvault/, modules/loganalytics/ — live
