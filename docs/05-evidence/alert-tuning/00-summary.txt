ROOT CAUSES IDENTIFIED
======================
1. PRIMARY: autoMitigate=false on ALL 5 alert rules
   - Azure Monitor Scheduled Query Rules v2 default: autoMitigate=false
   - Effect: once an alert fires, it generates a NEW instance every evaluation
     period (5 minutes) indefinitely, even when condition is no longer met
   - Evidence: 61+ "New" alert instances found, all at ~5-minute intervals,
     none in "Resolved" state, KQL queries returning Count=0 (no active events)

2. SECONDARY: alert-app-docker-failure window overlap
   - window_duration=PT10M + evaluation_frequency=PT5M
   - A single event caught in TWO consecutive overlapping windows → 2 emails/event

3. TERTIARY: "Stopped" in Docker KQL has_any
   - Matched graceful Docker restarts (normal operations), not just failures
   - Removed from has_any filter
