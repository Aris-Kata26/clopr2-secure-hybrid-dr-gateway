Issue: Two catch-all rules permit ANY database user to connect from:
  - 10.0.0.0/16 (entire on-prem subnet — ~65K addresses)
  - 10.200.0.0/30 (WireGuard tunnel)

Risk: Any valid PostgreSQL user (e.g., postgres superuser) with the correct
password could connect from any on-prem host or over the WireGuard tunnel —
not just appuser/appdb as intended.
