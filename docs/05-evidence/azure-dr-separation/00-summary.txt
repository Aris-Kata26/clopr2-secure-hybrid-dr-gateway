Weakness Addressed
------------------
During validated full-site failover (2026-03-15), the FastAPI app container ran
ON the DR DB VM (vm-pg-dr-fce) using --network host. DB + App were collocated
on the same Azure VM.

Improvement Delivered
---------------------
1. Terraform: optional Azure app VM (vm-app-dr-fce, Standard_B1s) in dr-fce env
   - Same subnet as DB VM (dr-mgmt-subnet 10.20.2.0/24)
   - Static private IP 10.20.2.20
   - Docker-only cloud-init (no WireGuard, no PostgreSQL)
   - Auto-shutdown at 23:00
   - Gated by enable_app_dr_vm = false (default) — deploy when ready

2. fullsite-failover.sh: new --app-vm <host> flag
   - FS-7 (separated): transfer image from DB VM to app VM, run with DB_HOST=<db-vm-private-ip>
   - FS-8/FS-9: health checks routed to app VM host
   - Without --app-vm: collocated behaviour unchanged (backward compatible)

3. fullsite-fallback.sh: new --app-vm <host> flag
   - H-2: checks app running on correct host
   - FB-1: stops app on correct host
   - Without --app-vm: collocated behaviour unchanged (backward compatible)
